# Code Skills

## Git Commit & Push Rules

- After every code update, commit and push immediately. Don't wait to be asked.
- Commit and push each file separately - no bulk commits.
- Each file gets its own commit and push. No grouping files together in a single commit.
- No Claude attribution. No Co-Authored-By. No "Generated with Claude Code". Nothing Claude-related in commits or pushes.
- Commits go under the developer's name only.
- Write clear, concise commit messages describing what changed.
